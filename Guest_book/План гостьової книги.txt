1. Планування функціональності
Користувачі мають можливість залишати свої повідомлення через форму (ім’я, email, текст повідомлення).
Повідомлення зберігаються в базі даних.
Повідомлення відображаються на сторінці у хронологічному порядку (новіші зверху).
Реалізувати базові перевірки на коректність введених даних (валідація).
Захист від XSS-атак (екранізація виведеного HTML).
(Опціонально) Сортування або фільтрація повідомлень.



2. Налаштування оточення
Перевір, що PHP та MySQL встановлені й працюють.
Ініціалізуй проєкт, створивши нову папку.
Налаштуй базу даних:
Створи таблицю для зберігання повідомлень (messages) із полями:
id (INT, PRIMARY KEY, AUTO_INCREMENT).
name (VARCHAR, наприклад, 100 символів).
email (VARCHAR, наприклад, 100 символів).
message (TEXT).
created_at (TIMESTAMP).
Перевір підключення до бази даних із PHP.



3. Створення базової структури
HTML-сторінка:
Форма для введення повідомлення (поля: ім'я, email, текст повідомлення, кнопка "Надіслати").
Місце для виведення списку повідомлень.
PHP-скрипт:
Обробка даних із форми (POST).
Додавання даних до бази (INSERT).
Виведення повідомлень із бази (SELECT).



4. Валідація даних
Перевіряй, чи всі поля заповнені.
Перевірка формату email.
Обмеження довжини тексту для ім'я та повідомлення.
Виведення повідомлень про помилки, якщо валідація не пройшла.



5. Виведення даних
Реалізуй запит до бази для отримання всіх повідомлень (SELECT * FROM messages ORDER BY created_at DESC).
Виведи список повідомлень у вигляді карток (ім'я, email, текст, дата).
Екрануй введений текст для захисту від XSS:
Наприклад, використовуючи htmlspecialchars().



6. Дизайн (опціонально)
Додай базове оформлення через CSS (Bootstrap або власні стилі).
Зроби повідомлення читабельними:
Відокремлені блоки для кожного повідомлення.
Дата та час у зрозумілому форматі (наприклад, Y-m-d H:i).



7. Покращення функціоналу (опціонально)
Сортування: Додай можливість вибору сортування (за датою чи іменем).
Пошук: Реалізуй пошук повідомлень за ключовими словами.
Валідація на клієнтській стороні: Використай JavaScript для перевірки введених даних перед відправкою.
Пагінація: Виводь обмежену кількість повідомлень на сторінку.



8. Тестування
Перевір усі можливі сценарії:
Введення коректних даних.
Залишення порожніх полів.
Введення великого тексту.
Використання спеціальних символів у полях.
Виправ помилки, які виникли.
